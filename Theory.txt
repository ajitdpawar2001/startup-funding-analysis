# Theory of the Startup Funding Analysis Project #

	This project is grounded in the Five Steps of Data Analysis, moving from raw data to actionable insights and communication.

1. Asking the Questions
	The project's goal was defined by key analytical questions, which guided the dashboard's design:

	What are the overall investment trends (Month-on-Month) in the Indian startup ecosystem?

	Which features (sectors, cities, investors) contribute most significantly to funding analysis?


2. Data Wrangling (Preprocessing)
	This step involved transforming the raw dataset into a format suitable for analysis.

	Gathering Data: Data was gathered from a static source (a CSV file).

	Cleaning and Assessing: Pandas was used to import the data and perform essential cleaning (data transformation). This involved:

	Converting the date column to the correct data type (a necessary assessment step).

	Handling potential missing data (NaN values) to ensure reliable filtering and aggregation across different columns.

	Feature Engineering: The investors column was manipulated (split and exploded) to allow for accurate aggregation of investment totals per unique investor, which is critical for the "Investor Deep-Dive" feature.

3. Exploratory Data Analysis (EDA)
	EDA was conducted to explore the data and derive the metrics needed for the dashboard.

	Bivariate/Multivariate Analysis: Grouping operations were performed to calculate statistics like total funding per sector, city, or investor.

	Plotting Graphs (Data Visualization): Various visualizations (bar charts, pie charts, line charts) were designed to represent the trends discovered.

4. Drawing Conclusions
	Descriptive statistics (like sum, max, and mean funding amounts) were calculated and used to draw clear conclusions about market performance (e.g., identifying top investors or sectors).

5. Communicating Results (Data Storytelling)
	Instead of relying on reports or slides, the final insights are communicated using an interactive web application built with the Streamlit framework. This provides a dynamic, user-friendly way to explore the data conclusions.
